## 문제

수빈이는 TV를 보고 있다. 수빈이는 채널을 돌리려고 했지만, 버튼을 너무 세게 누르는 바람에, 일부 숫자 버튼이 고장났다.

리모컨에는 버튼이 0부터 9까지 숫자, +와 -가 있다. +를 누르면 현재 보고있는 채널에서 +1된 채널로 이동하고, -를 누르면 -1된 채널로 이동한다. 채널 0에서 -를 누른 경우에는 채널이 변하지 않고, 채널은 무한대 만큼 있다.

수빈이가 지금 이동하려고 하는 채널은 N이다. 어떤 버튼이 고장났는지 주어졌을 때, 채널 N으로 이동하기 위해서 버튼을 최소 몇 번 눌러야하는지 구하는 프로그램을 작성하시오. 

수빈이가 지금 보고 있는 채널은 100번이다.

---

## 입력

첫째 줄에 수빈이가 이동하려고 하는 채널 N (0 ≤ N ≤ 500,000)이 주어진다. 둘째 줄에는 고장난 버튼의 개수 M (0 ≤ M ≤ 10)이 주어진다. 고장난 버튼이 있는 경우에는 셋째 줄에는 고장난 버튼이 주어지며, 같은 버튼이 여러 번 주어지는 경우는 없다.

---

## 출력

첫째 줄에 채널 N으로 이동하기 위해 버튼을 최소 몇 번 눌러야 하는지를 출력한다.

---

## 풀이

```python
n = int(input())
m = int(input())
breaks = []
if m > 0:
    breaks = list(map(int, input().split()))

ret = abs(n - 100)
for num in range(1_000_000):
    check = True
    for num_ in str(num):
        if int(num_) in breaks:
            check = False
            break
    if check:
        press_num = len(str(num)) + abs(n - num)
        if press_num < ret:
            ret = press_num

print(ret)
```

0부터 9까지의 버튼과 채널을 위 아래로 이동할 수 있는 +, - 버튼이 주어진다고 할 때, 100번 채널에서 해당 채널로 변경하기 위해 눌러야 하는 버튼 횟수의 최소값을 구하는 문제입니다.

단순하게 수빈이가 이동하려고 하는 채널의 N 범위가 0 ~ 500,000이므로 각 자릿수 별로 모든 버튼을 한번씩 눌러본다고 할 때 10^6번의 경우의 수가 생기므로 주어진 시간 제한인 2초 내로 가능하다고 판단해 브루트 포스 알고리즘을 사용하여 문제 풀이를 시도했습니다.

가장 먼저 단순한 방법으로는 100번 채널에서 시작해 해당 채널까지 + 혹은 - 버튼을 클릭해 이동하는 방법이 있습니다. 이를 가장 초기값으로 설정한 뒤, 0번부터 1,000,000까지 모든 숫자를 확인해보면서 해당 숫자에서 버튼을 최소한으로 눌러서 해당 채널로 변경할 수 있다면 해당 횟수로 갱신하는 과정을 거칩니다.

이 때 채널 N의 범위가 0 ~ 500,000인데 1,000,000까지 탐색하는 이유는 이동하고자 하는 채널이 500,000번 채널이고 고장난 버튼이 0, 1, 2, 3, 4, 5, 6, 7, 8이라면 999,999에서부터 출발하여 500,000까지 - 버튼만 눌러서 이동해야하는 테스트 케이스가 존재하기 때문입니다.

또 구현 시 주의할 점으로는 고장난 버튼이 없을 때는 고장난 버튼을 입력받지 않도록 주의해야 합니다.

모든 테스트 케이스에 대해 걸리는 시간이 일정하므로 첫번째 테스트 케이스만 통과한다면 주어진 시간 내에 문제 풀이에 성공할 수 있습니다.
