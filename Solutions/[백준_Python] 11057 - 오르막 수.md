## 문제

오르막 수는 수의 자리가 오름차순을 이루는 수를 말한다. 이때, 인접한 수가 같아도 오름차순으로 친다.

예를 들어, 2234와 3678, 11119는 오르막 수이지만, 2232, 3676, 91111은 오르막 수가 아니다.

수의 길이 N이 주어졌을 때, 오르막 수의 개수를 구하는 프로그램을 작성하시오. 수는 0으로 시작할 수 있다.

---

## 입력

첫째 줄에 N (1 ≤ N ≤ 1,000)이 주어진다.

---

## 출력

첫째 줄에 길이가 N인 오르막 수의 개수를 10,007로 나눈 나머지를 출력한다.

---

## 풀이

```python
n = int(input())
dp = [[0 for _ in range(10)] for _ in range(n)]

for i in range(n):
    for j in range(10):
        # base case
        if i == 0:
            dp[0][j] = 1
        else:
            # sum the number of (i-1) index digit cases that bigger than i index number.
            for k in range(j, 10):
                dp[i][j] += dp[i - 1][k]

print(sum(dp[-1]) % 10007)

```

자신보다 하나 작은 자리의 숫자는 자신보다 크거나 같아야 하는 규칙을 가진 문제입니다. 문제의 규칙을 보면 직감적으로 동적 계획법을 사용해야 한다고 느껴져 동적 계획법을 사용하여 문제 풀이를 성공하였습니다. 먼저 메모이제이션을 위한 배열로 0부터 9까지의 데이터를 저장할 리스트를 원소로 하는 n 길이의 dp 배열을 선언하였습니다.

가장 작은 기저 문제인 한 자리 수의 경우 0부터 9까지 모든 수가 한 가지 경우의 수 밖에 없으므로 1을 할당하고, 두 자리 수부터 하나 작은 자리의 수 중 자신보다 크거나 같은 수의 경우의 수의 합이 되므로 이에 맞춰 코드로 구현했습니다.

완전 탐색으로 구현 시에 중첩되는 부분 문제들이 매우 많기 때문에 이를 캐치하고 동적 계획법으로 구현해야 함을 알아채는 것이 중요한 문제였습니다.
